# 示例需求：单页待办（MVP + 扩展）

目标：在 `apps/web` 内实现一个**单页**待办应用，满足以下可验收条目。

## 功能（MVP 基线）

1. 用户可以在输入框输入文字并点击「添加」或按 Enter，将任务加入列表。
2. 每条任务展示文本与「完成」切换（checkbox 或按钮）；已完成任务有清晰视觉区分（删除线或降低对比度）。
3. 用户可以删除单条任务。
4. 提供「清空已完成」按钮，仅移除已完成项。
5. 列表为空时显示简短空状态提示（中文即可）。

## 功能（扩展 — 本轮迭代）

6. **编辑**：用户可对已输入的待办**修改正文**（行内编辑或独立编辑态均可），支持保存 / 取消，避免误触。
7. **实现日期**：每条待办可设置**截止日期**（按「日」粒度即可）；列表中展示日期，**已过期未完成**与**今日到期**需有视觉区分。
8. **归档与分类**：待办可归入**分类**（至少内置若干可选标签，如工作 / 生活 / 学习 / 收件箱）；支持**归档**（与「完成」不同：归档后进入归档区，可恢复）；主列表与归档区可切换查看。
9. **持久化**：刷新页面后待办仍存在（推荐 `localStorage`）。

## 技术约束

- 使用现有 Vite + React + TypeScript 栈；主要逻辑放在 `apps/web/src/App.tsx`（可拆小组件到同目录）。
- 样式可用 CSS 模块或 `App.css`，需有基本间距与可读性，避免纯默认「紫渐变卡片」模板感；鼓励有明确视觉主题（如偏科技感、年轻态）。

## 非目标

- 后端多用户同步、附件、子任务拆解非必须。
